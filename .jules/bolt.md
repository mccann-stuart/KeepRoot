## 2026-04-02 - Eliminate network waterfalls with Promise.all and D1 batching
**Learning:** Cloudflare Workers can experience significant latency when making sequential external network calls (e.g., fetching from D1 and then R2).
**Action:** When combining dependent database reads with independent R2 fetches (or other network I/O), aggressively execute them concurrently using `Promise.all()`. Furthermore, use `D1Database.batch()` to collapse multiple independent database queries into a single roundtrip to further flatten the waterfall. In this codebase, `getBookmark` in `backend/src/storage/bookmarks.ts` was refactored to fetch tags, images, and content metadata simultaneously.
