## 2024-05-14 - Batch D1 Database Reads
**Learning:** Cloudflare D1 and `workers-types` provide `D1Database.batch()` which allows sending multiple database reads in a single HTTP network roundtrip. In environments like Cloudflare Workers where database latency is network-bound, sequential `.prepare().first()` or `.all()` calls (like fetching a record and then its recent runs) can introduce significant N+1 network latency.
**Action:** When making multiple independent database queries that can be resolved simultaneously (e.g. fetching an entity and its associated metadata/runs by ID), combine them into a single `await env.DB.batch([ ... ])` call and destructure the results.
